Objectif du POC

Le dossier poc/ est une maquette haute-fidélité destinée à :

  • Valider l'interface et la navigation avant le développement
  • Présenter le projet sans infrastructure back-end
  • Servir de référence visuelle pendant le développement

Ce que le projet dynamic ajoute

  1. Persistance — les données survivent au rechargement de page
  2. Authentification — chaque bailleur a ses propres données cloisonnées
  3. Validation serveur — les formulaires sont validés côté Laravel
  4. Actions métier — mark-paid, archivage contrat, progression maintenance
  5. Composants réutilisables — zéro duplication HTML grâce aux composants Blade
  6. Rapports — agrégation de données réelles
  7. Upload de fichiers — stockage des documents sur le serveur
