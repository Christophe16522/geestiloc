Flux d'authentification

  1. L'utilisateur accède à / → redirection automatique vers /dashboard
  2. Si non authentifié → redirection vers /login (middleware auth)
  3. Inscription via /register (Laravel Breeze)
  4. Après login → accès à toutes les routes du groupe auth
  5. Toutes les données sont filtrées par user_id (isolation multi-utilisateurs)

Principes appliqués

  • SOLID — un contrôleur par ressource, responsabilité unique
  • DRY — composants Blade réutilisables pour tous les éléments d'UI récurrents
  • Scoped data — chaque query filtre sur user_id pour isoler les données par bailleur
